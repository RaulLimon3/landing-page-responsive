## 🌿 Git Workflow

This project was developed using a version-based branching strategy, where each branch represents a specific stage of the development:

```
main
├── v1-layout → Base HTML structure
├── v2-base-styles → Global styles and variables
├── v3-components → Reusable components (cards, buttons, etc.)
├── v4-sections → Full section development
└── v5-responsive → Responsive design implementation
```

Each version focuses on a specific layer of the project, allowing better control over progress and easier debugging.
